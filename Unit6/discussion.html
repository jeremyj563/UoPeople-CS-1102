Explanation of what the program does

  • Upon execution a small empty window is created with a title bar that is set to the text COME CLOSER
  • When the user hovers the mouse cursor over the window, the title text is set to I SEE YOU
  • If the mouse is clicked while still hovering over the window, the title text is set to LET GO
    • The title text is then set to WHEW when the mouse button is released
  • Immediately after clicking in the window, the title text is set to OUCH

Line-by-line explanation of how the program works

Class definition

  • The MouseWhisperer class extends (inherits from) the javax.swing.JFrame class
    • Any exposed code belonging to the JFrame class (and its ancestors) is available to the MouseWhisperer class
  • The MouseWhisperer class also implements the java.awt.event.MouseListener interface
    • MouseWhisperer must provide implementations for all member definitions belonging to the MouseListener interface (and its ancestors)

Private field definitions

  • The single private field serialVersionUID of type long is correctly declared using the final keyword making it a write-once field
  • The serialVersionUID is only useful as a unique identifier when serializing a class instance and therefore does not serve any purpose for the actual functionality of this example program

Class constructor

  • The MouseWhisperer() constructor calls the JFrame(String title) base class constructor using super() which in-turn creates a new instance of the java.awt.Frame class with the specified title
  • The java.awt.Window.setSize() setter is called passing in a width of 300 and a height of 100 which, of course, sets the size of the new window accordingly
  • The java.swing.JFrame.setDefaultCloseOperation() setter is called passing in the JFrame.EXIT_ON_CLOSE setting which causes the application to terminate once the window is closed
  • The java.awt.Component.addMouseListener() method is called which requires an object instance that implements the MouseListener interface as its sole arguement
    • The argument being passed in is this which is simply a reference to the current instance of the MouseWhisperer class, which is a valid implementation of the MouseListener interface
  • The java.awt.Window.setVisible() setter is called passing in true which obviously causes the window to be visible to the user

Event handler definitions and implementations

  • All five methods declared in the MouseListener interface are implemented
    • mouseClicked(MouseEvent e)
    • mousePressed(MouseEvent e)
    • mouseReleased(MouseEvent e)
    • mouseEntered(MouseEvent e)
    • mouseExited(MouseEvent e)
  • Each of these method implementations simply call the java.awt.Frame.setTitle() setter and pass in the appropriate title text string as described above
  • The setTitle() setter is made available to the MouseWhisperer class through the following inheritance hierarchy:
    • The MouseWhisperer class extends the javax.swing.JFrame class
    • The JFrame class extends the java.awt.Frame class
    • The Frame class contains the setTitle() method

Program entrypoint

  • The static MouseWhisperer.main() method simply creates a new instance of the MouseWhisperer class which causes the parameterless class constructor to be executed
